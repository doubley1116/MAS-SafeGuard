## 启动方式

在 PowerShell 中执行：

```powershell
powershell -ExecutionPolicy Bypass -File "C:\gpbell\grade2_2\Zero_Trust\frontend_showcase\start_showcase.ps1"
```

启动后访问：

```text
http://127.0.0.1:48317
```

## 组会 Demo 操作方法

| 步骤 | 操作 | 讲解重点 |
| --- | --- | --- |
| 1 | 执行 `start_showcase.ps1` 并打开 `http://127.0.0.1:48317` | 说明这是本地项目展示页，不依赖部署服务器 |
| 2 | 点击顶部导航 `动态演示` | 页面是单页分导航索引，客户不用上下滑动找功能 |
| 3 | 选择框架：`AutoGen / CrewAI / LangGraph` | 对应 `MAS/` 下三个多智能体框架 |
| 4 | 选择领域：电商、医疗、金融交易 | 对应不同业务风险场景 |
| 5 | 选择攻击类型 | 路径绕过、调用者伪装、语义注入、路由劫持、IPI、AiTM、感染式传播 |
| 6 | 优先选择 `Replay 稳定演示` | 组会推荐模式，不受 API Key、外部模型和网络状态影响 |
| 7 | 点击 `运行演示` | 展示预检卡片、运行实例看板、Agent 证据链和为什么拦截 |
| 8 | 点击 `载入生成结果` 或切到 `工作流审计` | 展示 workflow JSON、事件时间线、history 窗口和 blocked reason |
| 9 | 如环境完整，再切 `Live 真实运行 MAS` | 真实跑 MAS 脚本，依赖 Python 包和 `API_KEY / BASE_URL / MODEL` |
